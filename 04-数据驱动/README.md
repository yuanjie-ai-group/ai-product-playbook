# 数据驱动

> 没有数据支撑的产品决策就是拍脑袋

## 学习目标

建立数据驱动的决策习惯，能设计指标体系、分析用户行为、运行 A/B 测试。

## 子目录

### 指标体系/
- 核心指标 vs 虚荣指标
- 指标层级：北极星 → 一级指标 → 二级指标
- 数据看板设计
- 日报/周报中的数据解读

### AB测试/
- 实验设计基础：假设 → 方案 → 样本量 → 执行 → 分析
- 统计显著性理解
- 常见陷阱：辛普森悖论、多重比较、偷看数据
- AI 产品的 A/B 测试特殊性（非确定性输出）

### 用户行为分析/
- 漏斗分析：转化率在哪里断裂
- 路径分析：用户实际怎么使用产品
- 留存分析：留存曲线、cohort 分析
- 流失分析：用户为什么走了

### 数据工具/
- 埋点方案设计（事件 + 属性）
- 分析工具选型：Mixpanel / Amplitude / PostHog
- 自建 vs 第三方数据平台
- SQL 分析实战（工程师的优势技能）

## 工程师的数据优势

作为工程师，你比产品经理更有优势：
- 能直接写 SQL 查数据，不依赖数据团队
- 能理解数据管道和数据质量问题
- 能设计合理的埋点方案
- 能搭建自动化数据看板
