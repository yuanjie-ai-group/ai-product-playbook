# SEO 与 GEO 优化策略

> 搜索引擎优化（SEO）和生成式引擎优化（GEO）实操指南
>
> 创建日期：2026-03-30

## SEO vs GEO：核心区别

![SEO vs GEO 对比](../assets/01-seo-vs-geo.png)

| | SEO | GEO |
|---|---|---|
| **优化对象** | Google/Baidu 等传统搜索引擎 | ChatGPT、Perplexity、Google AI Overview 等 AI 搜索 |
| **核心目标** | 排名靠前 | 被 AI 引用/推荐 |
| **关键因素** | 关键词、外链、技术指标 | 内容权威性、结构化数据、可引用性 |
| **衡量标准** | 排名位置、点击率 | 被引用次数、品牌提及率 |

> **GEO 核心洞察**：AI 搜索引擎不按"排名"展示结果，而是直接生成答案并引用来源。目标是成为被引用的来源——内容的"可引用性"比"可点击性"更重要。

## 一、SEO 优化

### 技术 SEO 审计清单

- [ ] 所有页面有唯一的 `<title>` 和 `<meta description>`
- [ ] 正确的 `<h1>` - `<h6>` 层级结构
- [ ] `sitemap.xml` 配置完整且自动更新
- [ ] `robots.txt` 配置正确
- [ ] 图片有 alt 属性、使用 lazy loading、WebP 格式
- [ ] Core Web Vitals 达标（LCP < 2.5s, FID < 100ms, CLS < 0.1）
- [ ] 移动端适配良好
- [ ] HTTPS 全站启用

### 用 AI 工具辅助审计

```bash
# 扫描 SEO 问题
claude "审计项目的 SEO：meta tags、sitemap、robots.txt、结构化数据、图片 alt、性能"

# 批量生成落地页
claude "为每个产品生成 SEO 优化的落地页：语义化 HTML、结构化数据、内链、meta"

# 内容分析
claude "分析所有文章的关键词密度、标题结构、内链数量，输出 SEO 报告"
```

### 内容 SEO 策略

1. **关键词研究**：围绕核心业务挖掘长尾关键词
2. **内容矩阵**：技术博客 + 案例展示 + 行业洞察
3. **内链建设**：页面之间形成主题聚合（Topic Cluster）
4. **外链获取**：技术社区分享、开源项目、行业媒体投稿

## 二、GEO 优化

### 提升"可引用性"

AI 搜索引擎偏爱**结构清晰、有明确结论**的内容：

1. **开头直接回答** — 每篇前 50 字给出核心结论
2. **清晰层级** — H2/H3 结构化组织
3. **数据可视化** — 关键数据用表格或列表
4. **明确总结** — 结尾有总结段落
5. **FAQ 结构** — 添加 FAQPage Schema

### 结构化数据（Schema.org）

![Schema.org 结构化数据类型](../assets/07-schema-org-types.png)

| Schema 类型 | 用途 | 优先级 |
|---|---|---|
| `Organization` | 公司信息、Logo、联系方式 | 必须 |
| `Product` | 产品/服务信息 | 必须 |
| `FAQPage` | 常见问题 | 必须 |
| `HowTo` | 使用教程 | 推荐 |
| `Article` | 文章（含 author、date） | 推荐 |

### 建立"实体权威性"

需要创建的关键页面：

| 页面 | 目的 |
|------|------|
| `/about` | 公司介绍，建立实体信息 |
| `/glossary` | 行业术语表，建立领域专业性 |
| `/comparison` | 方案对比，回答用户对比问题 |
| `/pricing` | 清晰定价，AI 常被问价格 |
| `/cases` | 成功案例，增加可信度 |

### 监测 AI 搜索中的品牌表现

```bash
claude "写脚本每天向 Perplexity API 提问行业问题，检查是否提及我们品牌，记录并生成周报"
```

```
 问题    代码    文章    监测    优化
```

---

## 四、核心原则

> 你不是在优化给搜索引擎爬虫看的页面，而是在创建 AI 想要引用的信源。
> 内容的准确性、权威性、结构清晰度比关键词密度和外链数量更重要。
> 对于技术公司，发布高质量的技术方案文档本身就是最好的 GEO 策略。
