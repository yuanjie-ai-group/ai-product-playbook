# 团队与领导力

> 从个人贡献者（IC）到团队负责人到合伙人

## 学习目标

培养团队管理、跨部门协作、战略执行能力，逐步胜任合伙人角色。

## 子目录

### 团队搭建/
- 初创团队的招聘策略：找"对的人"而不是"最强的人"
- 团队文化建设：价值观落地
- 激励机制：股权、期权、成长空间
- 远程/混合办公团队管理

### 跨部门协作/
- 产研协同：工程师出身的独特优势
- 向上管理：与 CEO/投资人的沟通
- 向下赋能：帮助团队成员成长
- 冲突管理：产品与技术的常见分歧

### 战略执行/
- OKR 制定与跟踪
- 产品路线图规划
- 资源分配与优先级决策
- 复盘方法论

### 合伙人修炼/
- 全局视角：不只看自己负责的部分
- 决策框架：信息不完整时如何做判断
- 股权与治理：合伙人之间的权责划分
- 心态修炼：承受不确定性和压力

## 从工程师到合伙人的关键转变

| 维度 | 工程师 | 合伙人 |
|------|--------|--------|
| 关注点 | 代码质量、系统性能 | 公司发展、团队成长 |
| 成就感来源 | 解决技术难题 | 团队和产品的成功 |
| 时间分配 | 80% 写代码 | 30% 技术 + 30% 管理 + 40% 战略 |
| 决策方式 | 数据和逻辑 | 数据 + 直觉 + 判断力 |
| 责任范围 | 自己的模块 | 整条业务线甚至整个公司 |
