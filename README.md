# AI Product Playbook

> AI 产品全链路修炼手册：从工程师到合伙人的成长路径

## 为什么要做这个仓库

作为后端工程师出身，技术能力是基本盘。但在 AI 初创团队中，仅靠技术远远不够——你需要理解用户、驱动增长、做出商业判断、带领团队。

这个仓库记录从**工程师 → 产品 → 运营 → 商业 → 合伙人**的全链路成长过程，所有内容围绕 AI 产品实战展开。

## 成长路径

```
工程师思维          产品思维            商业思维           合伙人思维
───────────────────────────────────────────────────────────────────────
怎么实现？    →    该不该做？    →    能不能赚钱？   →   公司往哪走？
系统怎么跑    →    用户怎么用    →    市场怎么打     →   组织怎么建
技术最优解    →    体验最优解    →    商业最优解     →   战略最优解
```

## 学习路线图

### Phase 1：产品基本功（1-2个月）
- [ ] 完成 3 次用户访谈并输出用户画像
- [ ] 学会写 PRD（从模仿开始）
- [ ] 拆解 5 个 AI 产品的设计决策
- [ ] 建立"需求三问"习惯：给谁用？解决什么问题？怎么衡量成功？

### Phase 2：AI 产品专项（2-3个月）
- [ ] 总结 AI 产品的 5 大设计模式
- [ ] 梳理 AI 产品的 UX 挑战与解决方案
- [ ] 实践 Prompt 产品化（从内部工具开始）
- [ ] 建立 AI 产品的评估指标体系

### Phase 3：增长与运营（3-4个月）
- [ ] 设计产品的北极星指标和 AARRR 漏斗
- [ ] 跑通一个获客渠道（内容/社区/合作）
- [ ] 建立数据埋点和分析看板
- [ ] 完成首次 A/B 测试闭环

### Phase 4：商业思维（4-6个月）
- [ ] 梳理产品的商业模式和收入结构
- [ ] 制定并验证定价策略
- [ ] 完成竞争格局分析和市场定位
- [ ] 输出一份完整的商业计划书

### Phase 5：企业运营（持续）
- [ ] 建立财务看板（现金跑道、项目利润率、回款周期）
- [ ] 梳理合同模板库（开发合同、NDA、合伙人协议）
- [ ] 搭建客户管理流程（商机漏斗 → 交付 → 维护）
- [ ] 完成商标注册和软件著作权申请

### Phase 6：领导力进阶（持续）
- [ ] 建立跨部门协作机制
- [ ] 制定团队 OKR 和产品路线图
- [ ] 培养至少 1 个能独当一面的下属
- [ ] 形成自己的决策框架

## 核心原则

1. **实战优先** —— 所有内容必须结合 yuanjie 实际产品，不做纯理论
2. **工程师优势** —— 利用技术背景（懂成本、懂可行性、懂数据），而不是抛弃它
3. **回头成本思维** —— 在哪里投入深度学习，取决于做错了改的成本有多高
4. **输出倒逼输入** —— 每学一个主题，必须输出一篇实战总结
