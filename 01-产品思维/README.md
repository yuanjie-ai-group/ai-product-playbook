# 产品思维

> 从"怎么实现"到"该不该做"

## 学习目标

掌握产品经理的核心方法论，培养用户视角和产品判断力。

## 子目录

### 用户研究/
- 用户访谈方法与技巧
- 用户画像构建
- 用户旅程地图（User Journey Map）
- 用户反馈收集与分析

### 需求分析/
- 需求挖掘：表面需求 vs 深层需求
- 需求优先级：RICE、ICE、MoSCoW 框架
- PRD 编写方法
- 需求评审中的"三问"：给谁用？解决什么问题？怎么衡量？

### 产品设计/
- 信息架构设计
- 交互设计基础
- MVP 设计与快速验证
- 产品原型工具使用

### 竞品分析/
- 竞品分析框架（功能、体验、商业、技术）
- 竞品对比表模板
- 从竞品中发现机会点

## 工程师视角的产品思维

作为工程师，你已经具备的优势：
- **逻辑严密** —— 能发现需求中的逻辑漏洞
- **边界思考** —— 能想到产品经理忽略的边界情况
- **成本意识** —— 能评估方案的技术成本，提出更优替代方案
- **系统思维** —— 能看到单个功能对整体系统的影响

需要补足的能力：
- **同理心** —— 从用户的情绪和感受出发
- **取舍能力** —— 不是"都能做"而是"该做什么"
- **表达能力** —— 把想法变成别人能理解的方案
